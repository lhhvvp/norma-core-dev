# ElRobot Follower Simulation Worlds

This directory contains simulation worlds for the ElRobot follower arm.

## Files

- `elrobot_follower.world.yaml` — **human-editable** manifest describing the world, robots, actuators, sensors, and their capabilities. This is the single source of truth for sim-time metadata.
- `elrobot_follower.xml` — **generated** MJCF file (do not edit; regenerate via `make regen-mjcf`).
- `gen.py` — script that reads the `world.yaml` manifest plus the referenced URDF and produces MJCF.

## Workflow

1. Edit `elrobot_follower.world.yaml` (or the referenced URDF)
2. Run `make regen-mjcf`
3. Commit all three files together (yaml + xml) so downstream CI sees a consistent snapshot

## MJCF Source Hash

The generated `elrobot_follower.xml` contains an XML comment like:
```
<!-- norma-sim: generated by gen.py
     source_hash=sha256:7c9e6679... (urdf + world.yaml) -->
```

At runtime, `norma_sim` verifies that `sha256(urdf + world.yaml)` matches the hash
embedded in the MJCF. Mismatch means the manifest/URDF were edited but gen.py was
not re-run; `norma_sim` will fail fast with an actionable error.

## Why two files instead of one?

The manifest (`world.yaml`) describes capabilities, display names, and units.
The MJCF describes the physics scene MuJoCo actually simulates. gen.py derives
MJCF from URDF + manifest because it's cheaper to regenerate than to hand-edit
MJCF every time hardware changes.
